# 🛍 Digital Shop Bot

Телеграм-магазин цифровых товаров: каталог, корзина, промокоды, оплата через **Telegram Stars** и мгновенная выдача доступов.

---

## Возможности

**Для покупателя**
- 🛍 Каталог с категориями
- 🛒 Корзина: изменение количества, очистка, пересчёт суммы
- 🎟 Промокоды: скидка в процентах или фиксированная
- ⭐️ Оплата через Telegram Stars — без внешнего платёжного провайдера
- ⚡️ Мгновенная выдача доступов после оплаты
- 📦 История заказов с повторным получением доступов

**Для администратора**
- 📊 Статистика: выручка за день / неделю / всё время, число покупателей, топ товаров, баланс бота
- ➕ Добавление товаров через диалог
- 🎟 Создание промокодов с лимитом применений
- 📦 Просмотр всех заказов
- 🔔 Уведомление о каждой покупке

## Стек

- **Python 3.11**
- **aiogram 3.x** — асинхронный фреймворк Telegram Bot API
- **aiosqlite** — асинхронный SQLite
- **Telegram Stars (XTR)** — платежи

---

## Технические решения

**Полный цикл платежа.** `send_invoice` → `pre_checkout_query` → `successful_payment`. Заказ фиксируется в БД только после подтверждения оплаты Telegram, товар выдаётся в том же хендлере.

**FSM.** Оформление заказа (e-mail → промокод → счёт), добавление товара и создание промокода — конечные автоматы на `StatesGroup` с валидацией каждого шага и выходом по `/cancel`.

**Схема БД.** 5 таблиц: `products`, `cart`, `orders`, `order_items`, `promocodes`. Позиции заказа сохраняются с ценой на момент покупки — изменение цены товара не искажает историю.

**Авторизация.** Права администратора проверяются единым guard-хендлером на входе в админку, а не копипастой в каждом обработчике. Доступ к чужому заказу невозможен: `callback_data` подделывается на клиенте, поэтому владелец заказа проверяется запросом к БД.

**Ограничения Stars.** Минимальная сумма счёта — 1 ⭐, поэтому скидка по промокоду автоматически урезается так, чтобы к оплате оставалась хотя бы одна звезда.

## Запуск

```bash
git clone https://github.com/USERNAME/telegram-shop-bot.git
cd telegram-shop-bot

pip install -r requirements.txt

cp .env.example .env   # заполнить своими значениями
python main.py
```

### Переменные окружения

| Переменная | Описание | Где взять |
|---|---|---|
| `BOT_TOKEN` | Токен бота | [@BotFather](https://t.me/BotFather) → `/newbot` |
| `ADMIN_ID` | Telegram ID администратора | [@userinfobot](https://t.me/userinfobot) |

При первом запуске создаётся `shop.db` и заполняется демо-товарами.
